# Лабораторная работа №2: Коллекция объектов StudentCollection

**Коллекция** — это структура данных, предназначенная для хранения и управления группой однотипных объектов. В объектно-ориентированном программировании коллекции предоставляют удобный интерфейс для работы с множеством объектов, скрывая детали внутренней реализации.

### Преимущества использования коллекций
1. **Инкапсуляция** — скрывает внутреннюю структуру хранения данных       
2. **Контроль доступа** — позволяет проверять корректность добавляемых объектов     
3. **Удобство использования** — предоставляет понятные методы для работы с группой объектов           
4. **Расширяемость** — легко добавлять новые методы работы с коллекцией           

## Предметная область
Используется тот же вариант, что и в лабораторной работе №1 — "Образование". Основная сущность — Студент (Student).           

Коллекция StudentCollection управляет группой студентов, позволяя:            
- Добавлять новых студентов в группу            
- Удалять студентов из группы           
- Искать студентов по различным критериям           
- Сортировать студентов по разным параметрам            
- Получать подмножества студентов (например, только активных)

## Коллекция StudentCollection
### Назначение коллекции StudentCollection
Коллекция `StudentCollection` предназначена для:            
- Хранения объектов класса `Student` (из лабораторной работы №1)            
- Управления группой студентов (добавление, удаление, поиск)            
- Предоставления удобного доступа к элементам коллекции           
- Реализации операций фильтрации и сортировки           
- Обеспечения целостности данных (проверка типов, защита от дубликатов)           

## Атрибуты и методы
**Метод** — это функция, принадлежащая классу, которая определяет поведение объектов этого класса. Методы позволяют взаимодействовать с объектом, изменять его состояние или получать информацию о нем.         

Основные методы управления          
>add(student) добавляет студента в коллекцию            
>>Проверяет, что добавляемый объект — действительно студент           
>>Проверяет, нет ли такого студента в коллекции           
>>Добавляет студента в конец списка           
>           
>remove(student) удаляет студента из коллекции            
>>Находит и удаляет указанного студента из коллекции            
>>Вызывает ошибку, если студента нет в коллекции            
>           
>remove_at(index) удаляет студента по индексу           
>>Удаляет студента, находящегося на позиции index           
>>Возвращает удаленного студента            
>>Проверяет, что индекс существует            
>
>get_all() получает список всех студентов
>>Возвращает внутренний список студентов            
>>Позволяет получить доступ ко всем элементам           

## Сортировка
**Сортировка** — это процесс упорядочивания элементов коллекции по определенному критерию. В коллекции StudentCollection реализована универсальная сортировка через метод sort(key), который принимает функцию-ключ.            
>sort(key) отсортировывает коллекцию по ключу           

Лямбда-функции используются для создания простых функций-ключей:            
>lambda st: st.surname — сортировка по фамилии            
>lambda st: st.curse — сортировка по курсу            
>lambda st: -st.gpa — сортировка по GPA в обратном порядке            

## Фильтрация
**Фильтрация** — это процесс отбора элементов коллекции, удовлетворяющих определенному условию. Метод get_active() демонстрирует принцип фильтрации, возвращая новую коллекцию с отфильтрованными объектами.            
Методы поиска и фильтрации            
>find_by_surname(surname) находит студентов по фамилии            
>find_by_course(course)	находит студентов по курсу            
>get_active() получает коллекцию активных студентов           

## Магические методы
Класс StudentCollection реализует следующие магические методы для удобной работы:           
>`__str__()` Определяет строковое представление объекта для пользователя. Вызывается при print()            
>`__len__()` Возвращает длину коллекции. Вызывается функцией len()            
>`__iter__()` Возвращает итератор для обхода коллекции. Позволяет использовать цикл for           
>`__getitem__(index)` Обеспечивает доступ к элементам по индексу. Поддерживает синтаксис collection[index]            

## Бизнес-методы коллекции
1. Фильтрация активных студентов            
`get_active` возвращает новую коллекцию активных студентов            

Особенности:            
- Создает новую коллекцию (не изменяет исходную)            
- Включает только студентов со статусом is_active = True            
- Сохраняет все методы исходной коллекции           

2. Поиск по фамилии           
`find_by_surname` поиск студентов по фамилии            

Особенности:            
- Возвращает список (не коллекцию)            
- Может вернуть несколько студентов с одинаковой фамилией           
- При отсутствии результатов возвращает пустой список           

3. Поиск по курсу           
`find_by_course` поиск студентов по курсу           

Особенности:            
- Ищет студентов, обучающихся на определенном курсе           
- Возвращает список найденных студентов           

4. Сортировка           
`sort` сортировка коллекции по ключу            

Особенности:            
- Сортирует коллекцию на месте (изменяет исходную)            
- Принимает функцию-ключ для определения порядка            
- Поддерживает сортировку по любому атрибуту
